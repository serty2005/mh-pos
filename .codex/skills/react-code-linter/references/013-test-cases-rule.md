# 🧪 TEST CASES RULE (CRITICAL)

## ❗ Absolute rule

Every solution must include test cases.

## ✅ Mandatory coverage

### ✅ Main scenarios

- standard behavior

### ⚠️ Edge cases

- empty values
- null / undefined
- boundary values

### ❌ Negative scenarios

- API errors
- invalid data
- failures

## Format

### 🧪 Test cases

#### ✅ Happy path

- ...

#### ⚠️ Edge cases

- ...

#### ❌ Negative cases

- ...

## ❌ Forbidden

- writing a solution without test cases
- ignoring edge cases
- testing only the happy path

## 🧠 Principle

If it cannot be tested, it cannot be considered reliable.
